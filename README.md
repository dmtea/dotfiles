# Dotfiles

GNU Stow packages for Ubuntu 24.04+ (including Pop!_OS) laptop + bootstrap script.

## Structure

```
~/dotfiles/               ← clone here, stow root
├── install.sh            ← one-liner entry point (curl | bash)
├── scripts/
│   ├── bootstrap.sh      ← full machine provisioning (13 modules)
│   ├── modules/          ← individual bootstrap modules
│   ├── setup.md          ← walkthrough and troubleshooting
│   └── bin/              ← user scripts (installed to ~/.local/bin)
│       └── bt-toggle.sh
├── docs/
│   └── research/         ← investigations and solutions
├── zsh/                  ← stow packages
├── kitty/
├── nvim/
├── tmux/
├── opencode/
└── archive/              ← archived configs (not used in bootstrap)
```

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/dmtea/dotfiles/main/install.sh | bash
```

This clones the repo, creates a config from the example, and runs the full bootstrap. You will be prompted for `GIT_USER_NAME` and `GIT_USER_EMAIL` if not set in `bootstrap.conf`.

All output is logged to `/tmp/bootstrap-<timestamp>.log`.

### Manual (alternative)

```bash
git clone https://github.com/dmtea/dotfiles.git ~/dotfiles
cd ~/dotfiles/scripts/
cp bootstrap.conf.example bootstrap.conf
# edit bootstrap.conf with your name/email
./bootstrap.sh
```

## Stow Packages

Applied in step 1b (early stow) and finalized in step 9:

```sh
stow zsh       # .zshrc, .fzf.zsh, starship.toml
stow kitty     # kitty terminal config
stow opencode  # opencode CLI config + oh-my-openagent + skills + PATH
stow nvim      # Neovim config
stow tmux      # tmux config + TPM
```

`.gitconfig` is generated by `setup-git.sh` (not a stow package). `archive/` exists but is not stowed during bootstrap.

## Post-Bootstrap

After the script finishes:

1. **Generate age key**: `mkdir -p ~/.config/sops/age && age-keygen -o ~/.config/sops/age/keys.txt`
2. **Save the public key** to `.sops.yaml` in your project. Save the private key in Vaultwarden!
3. **Log out and back in** -- required for zsh default shell + Docker group

If `VW_URL` is set in `bootstrap.conf`, Vaultwarden is configured automatically during bootstrap. Manual `bw config server` + `bw login` is only needed for non-Vaultwarden setups.

## Requirements

- **OS**: Ubuntu 24.04+ (including Pop!_OS)
- **Architecture**: amd64 (x86_64)
- **Internet**: Required
- **Sudo**: Required (script calls sudo internally — do NOT run as root)
