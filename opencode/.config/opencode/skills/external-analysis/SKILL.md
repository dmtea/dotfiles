# External AI Analysis Protocol

**Scope:** global
**Trigger:** после значимых изменений AGENTS.md (MINOR/MAJOR бамп), по запросу создателя, или по напоминанию агента

---

## Когда использовать

1. **Hook после бампа AGENTS.md** — агент напоминает создателю:
   - MINOR бамп: «Существенные изменения в проекте. Рекомендуется внешний анализ. Запустить?»
   - MAJOR бамп: «Критические изменения в проекте. Настоятельно рекомендуется внешний анализ. Запустить?»
2. **По запросу создателя** — в любой момент
3. **Не чаще 1 раза в день** — если изменения мелкие, накопить

---

## Ключевой принцип: независимость анализа

**Предыдущие AI-анализы НЕ отправляются новым моделям.** Это критическое правило методологии.

Причины:
- **Якорение** — модель соглашается с уже высказанными позициями вместо формирования собственных
- **Слепота к уникальному** — «другие уже нашли X, не буду повторять» → пропускает важное
- **Фиктивная конвергенция** — 5/5 consensus, но 4 из 5 читали друг друга → не независимо
- **Groupthink** — через 3-4 итерации все модели думают одинаково, ценность нового анализа → 0

**Два режима отправки:**

| Режим | Что отправляется | Когда использовать |
|-------|-----------------|-------------------|
| **Independent** (по умолчанию) | AGENTS.md + docs/ **без** archive/ | Первичная отправка каждой модели — для независимого мнения |
| **Comparative** | AGENTS.md + docs/ + archive/ai_analysis/*_think.md (без digest!) | Повторный вопрос: «Что из предыдущих анализов других моделей ты считаешь неверным?» |

## Полный протокол (5 шагов)

### Шаг 1: Подготовка файлов для внешних моделей

Агент создаёт **один конкатенированный файл** для отправки:

```bash
tmp/PIP-project-for-analysis.md
```

**Содержание (только проектные документы):**
1. AGENTS.md — главный документ
2. docs/contacts-and-resources.md
3. docs/research/INDEX.md + все R##-файлы
4. docs/ideas/INDEX.md + все I##-файлы

**НЕ включаются:**
- archive/ (предыдущие анализы — см. принцип независимости)
- tmp/ (черновики, предложения)
- .git/, node_modules/ и т.д.

Формат: один Markdown файл с разделителями `=== SECTION ===` и `--- FILE: path ---`.

Агент создаёт файл автоматически при запуске протокола. Создатель скачивает/копирует и отправляет вручную.

**Промпт-шаблон** — см. секцию ниже.

### Шаг 2: Сбор сырых анализов

Создатель вставляет каждый ответ в файл:
```
archive/YYYY-MM-DD/ai_analysis/{model}_think.md
```

Если какие-то модели недоступны — пропускаем. Минимум 2 модели для валидного анализа.

### Шаг 3: Собственный аудит агента (параллельно с шагом 2)

Агент независимо анализирует текущий AGENTS.md на:
- 🔴 Критические проблемы — противоречия, нарушения красных линий (R1–R7)
- 🟡 Рекомендации — что добавить/изменить/улучшить
- 🟢 Сильные стороны — подтверждение правильных решений
- ❓ Вопросы к создателю — что требует уточнения
- 💡 Уникальные инсайты — неочевидные находки

Результат — в формате дайджеста (шаг 4, включается как колонка «Internal»).

### Шаг 4: Синтез → Дайджест

Агент прочитывает ВСЕ `_think.md` файлы + свой аудит и создаёт:

**Файл:** `archive/YYYY-MM-DD/ai_analysis/digest.md`

**Структура дайджеста:**

```markdown
# External Analysis Digest — YYYY-MM-DD

## Context
- AGENTS.md version: X.Y.Z
- Models analysed: ChatGPT, Claude, Grok, ZAI, Internal
- Trigger: [MINOR/MAJOR бамп | запрос создателя]
- Mode: independent / comparative

## Convergence Matrix

| Topic | ChatGPT | Claude | Grok | ZAI | Internal | Consensus? |
|-------|---------|--------|------|-----|----------|------------|
| ...   | ...     | ...    | ...  | ... | ...      | ✅/❌       |

## Convergent Points (consensus 3+)
### 1. [Topic]
- What models said: ...
- Internal assessment: ...
- **Action:** [accept / reject / discuss]

## Unique Insights (1 model only)
### 1. [Topic] — from [Model]
- What they said: ...
- **Action:** [accept / reject / discuss]

## Critical Problems Found
### 1. [Problem]
- Source: [which models flagged it]
- Severity: [critical / significant / minor]
- **Action:** [fix now / schedule / dismiss with reason]

## Recommendations (prioritized)
1. [Rec] — priority P0/P1/P2 — source: [models]

## Questions for Creator
1. [Question] — from [model]

## Disposition
| # | Recommendation | Verdict | Reason |
|---|---------------|---------|--------|
| 1 | ... | ✅ accepted | ... |
| 2 | ... | ❌ rejected | ... |
| 3 | ... | 🔄 discussed | ... |

## Integration Log
| Date | Action | AGENTS.md |
|---|---|---|
| ... | ... | ... |
```

### Шаг 5: Интеграция в AGENTS.md

Интерактивный проход по дайджесту с создателем:
- Каждая рекомендация — подтверждение/отклонение/уточнение
- Принятые — интегрируются в AGENTS.md
- Отклонённые — записываются в disposition таблицу с причиной
- Commit message ссылается на конкретные пункты дайджеста

## Промпт-шаблон для отправки внешним AI

### Independent mode (по умолчанию)

```
Проанализируй документ проекта ПИП (Персональный Интеллектуальный Партнёр).

Проект находится на стадии PRE-IMPLEMENTATION. Код писать рано — только исследования, спецификации, анализ.

Ответь в следующем формате:

🔴 КРИТИЧЕСКИЕ ПРОБЛЕМЫ
Противоречия между разделами, нарушения логики, конфликтующие статусные маркеры ([РЕШЕНО] vs [ОБСУЖДАЕТСЯ]), нарушения красных линий (R1–R7).

🟡 РЕКОМЕНДАЦИИ
Что добавить, изменить, усилить, предусмотреть. Конкретные предложения с обоснованием.

🟢 СИЛЬНЫЕ СТОРОНЫ
Что сделано хорошо и не требует изменений. Подтверждение правильных решений.

❓ ВОПРОСЫ К СОЗДАТЕЛЮ
Что требует уточнения. Неоднозначные места. Допущения которые нужно подтвердить.

💡 УНИКАЛЬНЫЕ ИНСАЙТЫ
Неочевидные находки, которые другие аналитики могут пропустить. Нестандартные перспективы. Аналогии из других доменов.

Пожалуйста, будь конкретен и честен. Если видишь проблему — называй конкретные разделы. Если хвалишь — объясни почему. Если не уверен — так и скажи.
```

### Comparative mode (опциональный второй проход)

```
Перед тобой проект ПИП и анализы от других AI-моделей.

Твоя задача — НЕ повторить их выводы, а найти:
1. С чем ты категорически НЕ согласен — и почему
2. Что все остальные пропустили — слепые зоны
3. Где предыдущие аналитики ошиблись — конкретные примеры

Будь максимально критичен. Конструктивная критика ценнее согласия.
```

## Правила для агента

1. **Никогда не пропускай шаг 3** (собственный аудит) — внешние модели не заменяют внутренний анализ
2. **Дайджест сохраняется в архив** — не в tmp, не удаляется
3. **Disposition обязателен** — каждая рекомендация получает вердикт с причиной
4. **Commit message** — содержит ссылку на дайджест и перечень интегрированных пунктов
5. **Минимум 2 модели** — одиночный анализ не даёт конвергенции
6. **Напоминание о хуке** — при MINOR/MAJOR бампе AGENTS.md агент обязан напомнить о внешнем анализе, но не навязывать
7. **Частоту определяет создатель** — агент только напоминает
8. **НЕ включай предыдущие анализы** в файл для новых моделей (принцип независимости)
9. **Создавай конкатенированный файл автоматически** на шаге 1 — создатель не должен собирать вручную
10. **Integration Log** — добавляется в конец дайджеста для отслеживания что реально интегрировано
