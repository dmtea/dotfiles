# Delegation Example

Complete example of delegating a test to a sub-agent using pane-tester.

## Overview

Parent agent delegates test execution to a fast sub-agent. Parent monitors progress and receives final results.

## Parent Agent Code

```typescript
// 1. Prepare test artifacts
const cleanupScript = `
#!/bin/bash
echo "Cleaning up..."
rm -f /tmp/test_*.tmp
echo "Cleanup complete"
`;

await write_file("/tmp/cleanup.sh", cleanupScript);

// 2. Create test plan
const testPlan = `
## TASK
Execute the following test plan EXACTLY. Do NOT deviate.

## TEST PLAN (TODOS)
- [ ] Create tmux pane
- [ ] Run: echo "Step 1: Initialize" && sleep 2 && echo "STEP1_OK"
- [ ] Verify: expect "STEP1_OK"
- [ ] Run: test -f /etc/hosts && echo "HOSTS_OK"
- [ ] Verify: expect "HOSTS_OK"
- [ ] Run: cat /etc/hostname
- [ ] Verify: output is not empty
- [ ] Run cleanup script: /tmp/cleanup.sh
- [ ] Kill pane

## OUTPUT REQUIREMENTS
- Save each step output to: /tmp/pane-tester-{AGENT_ID}/step_N.txt
- On ANY error: immediately save output and report FAILURE
- On success: report SUCCESS with summary

## RESULT FORMAT
End your response with exactly:
\`\`\`
RESULT: SUCCESS | FAILURE
ITERATION: N
STEPS_COMPLETED: N
STEPS_TOTAL: N
LAST_STEP: step_name
ERROR_IF_ANY: error_message_or_none
LOGS_DIR: /path/to/logs
SUMMARY_FILE: /path/to/summary.md
\`\`\`

## CRITICAL RULES
1. Follow plan EXACTLY - no improvisation
2. Use exact commands provided - no modifications
3. Wait for each command to complete before next
4. Capture output after each step
5. On failure: STOP immediately, report, do NOT retry or debug
`;

// 3. Delegate to sub-agent
const taskId = task(
  category="quick",
  load_skills=["pane-tester"],
  run_in_background=true,
  description="Test my script",
  prompt=testPlan
);

// 4. Watchdog monitoring
const statusFile = `/tmp/pane-tester-${taskId}/status.json`;
let lastStep = '';
let lastStepTime = Date.now();

for (let waited = 0; waited < 300; waited += 30) {
  await sleep(30);
  
  try {
    const status = JSON.parse(await read_file(statusFile));
    console.log(`[${status.timestamp}] ${status.steps_completed}/${status.steps_total} - ${status.step_name}`);
    
    if (status.status === 'complete') {
      console.log("Test passed!");
      break;
    }
    if (status.status === 'failed') {
      console.log("Test failed at:", status.step_name);
      break;
    }
    
    // Check for stuck
    if (status.step_name !== lastStep) {
      lastStep = status.step_name;
      lastStepTime = Date.now();
    } else if ((Date.now() - lastStepTime) / 1000 > 180) {
      console.log("WARNING: Stuck on", status.step_name);
    }
  } catch (e) {
    console.log("Waiting for sub-agent...");
  }
}

// 5. Get final result
const result = await background_output(taskId);
console.log(result);
```

## Sub-Agent Code (Auto-Generated by task())

The sub-agent receives the test plan and executes it:

```bash
# Sub-agent automatically:
# 1. Generates AGENT_ID
# 2. Creates OUTPUT_DIR
# 3. Creates tmux pane

# Step execution
PANE_ID=$(tmux split-window -v -P -F "#{pane_id}")

# Step 1
tmux send-keys -t ${PANE_ID} "echo 'Step 1: Initialize' && sleep 2 && echo 'STEP1_OK'" Enter
sleep 5
OUTPUT=$(tmux capture-pane -p -t ${PANE_ID} -S -20)
echo "${OUTPUT}" > ${OUTPUT_DIR}/step_1.txt
echo "${OUTPUT}" | grep -q "STEP1_OK" || { echo "FAIL at step 1"; exit 1; }

# Update status
echo '{"step": 1, "status": "in_progress", "step_name": "init"}' > ${STATUS_FILE}

# Step 2
tmux send-keys -t ${PANE_ID} "test -f /etc/hosts && echo 'HOSTS_OK'" Enter
sleep 3
OUTPUT=$(tmux capture-pane -p -t ${PANE_ID} -S -20)
echo "${OUTPUT}" > ${OUTPUT_DIR}/step_2.txt
echo "${OUTPUT}" | grep -q "HOSTS_OK" || { echo "FAIL at step 2"; exit 1; }

# Continue with remaining steps...

# Final cleanup
tmux send-keys -t ${PANE_ID} "/tmp/cleanup.sh" Enter
sleep 5
tmux kill-pane -t ${PANE_ID}

# Generate summary
echo "# Test Summary\n\nResult: SUCCESS" > ${OUTPUT_DIR}/summary.md
```

## Result Handling

```typescript
// Parse sub-agent result
if (result.includes("RESULT: SUCCESS")) {
  console.log("✅ All tests passed!");
  // Continue with confidence
} else {
  // Extract error details
  const failedStep = result.match(/LAST_STEP: (.+)/)?.[1];
  const logsDir = result.match(/LOGS_DIR: (.+)/)?.[1];
  
  console.log(`❌ Test failed at: ${failedStep}`);
  console.log(`Logs: ${logsDir}`);
  
  // Read error details
  const errorLog = await read_file(`${logsDir}/error.txt`);
  console.log(errorLog);
  
  // Decision: retry, fix, or abort
}
```

## Parallel Testing

```typescript
// Run multiple tests in parallel
const tests = [
  { name: "test_a", plan: planA },
  { name: "test_b", plan: planB },
  { name: "test_c", plan: planC },
];

const taskIds = tests.map(t => 
  task(
    category="quick",
    load_skills=["pane-tester"],
    run_in_background=true,
    description=t.name,
    prompt=t.plan
  )
);

// Collect all results
const results = await Promise.all(
  taskIds.map(id => background_output(id))
);

// Analyze
const passed = results.filter(r => r.includes("RESULT: SUCCESS")).length;
console.log(`${passed}/${tests.length} tests passed`);
```

## Key Points

1. **Parent agent** — orchestration, decisions, result handling
2. **Sub-agent** — strict execution, no improvisation
3. **Watchdog** — progress monitoring, stuck detection
4. **Result format** — structured for parsing
5. **Parallel** — multiple tests simultaneously
